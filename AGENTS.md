# AGENTS.md

このプロジェクトは、腕立てメニューをランダム抽選して記録する静的Webアプリです。

Codexがこのリポジトリで作業するときは、以下のルールに従ってください。

## 基本方針

- 作業前に、変更方針と影響範囲を簡潔に説明する。
- ユーザーから明示的な許可があるまで、不要なファイル作成や大きな構成変更をしない。
- 変更範囲はできるだけ小さく保つ。
- 既存のHTML/CSS/JavaScript構成を尊重し、不要にフレームワークやビルドツールを追加しない。
- 関係のないリファクタリングや整形変更は避ける。
- 既存のGit変更がある場合は、勝手に戻さない。

## アプリ仕様上の注意

- 当日の記録は `localStorage` に保存されている。
- `localStorage` の既存データ互換性を壊さない。
- 保存キーや保存データ構造を変更する場合は、既存データの移行または後方互換性を必ず考慮する。
- 種目、回数、履歴、設定の扱いを変更する場合は、画面表示とコピー結果への影響を確認する。
- 日付ごとの保存挙動を変える場合は、ユーザーに影響を説明する。

## UI変更時の注意

- モバイル表示を前提に、狭い画面でも文字やボタンが崩れないようにする。
- 既存のシンプルな静的アプリ構成を保つ。
- 見た目の変更は必要な範囲に限定する。
- 日本語UIの文言は、現在の自然なトーンに合わせる。

## 動作確認

変更後は、確認した内容をユーザーに説明する。

最低限、以下を確認する。

- ブラウザで `index.html` またはローカルサーバーを開けること
- 「次を引く」で種目が抽選されること
- 残り種目、実施済み種目、総回数、順番ログが更新されること
- ページ再読み込み後も当日の記録が保持されること
- リセット、コピー、設定チェックボックスが必要に応じて動作すること

ローカルサーバーで確認する場合は、プロジェクト直下で以下を使う。

```sh
python3 -m http.server 8000
```

## 変更後の報告

作業完了時は、以下を簡潔に報告する。

- 変更したファイル
- 変更内容の要点
- 実施した確認方法
- 未確認事項や注意点があればその内容
